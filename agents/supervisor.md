# 监督逻辑定义

被 `simulator.md` 引用，定义对话过程中的实时监督规则。

---

## 信息充分性检查

每轮护士回复后，对比已收集信息与标注病例中的真实信息，判断是否足以完成科室分诊。

### 判断标准

| 信息项 | 重要性 | 判断标准 |
|--------|--------|---------|
| 主诉 | 必须 | 核心症状和持续时间已被提及 |
| 现病史 | 重要 | 症状发展、伴随症状大致被了解 |
| 既往史 | 非必须 | 高血压/糖尿病/心脏病/手术外伤史，有则加分 |
| 药物过敏史 | 非必须 | 被问到则加分 |

### 判断原则

- 信息大致全面即可，不要求与标注病例逐字一致
- 允许表述差异和轻微遗漏
- 主诉和现病史是核心，收集到这两项基本就够
- 患者情绪着急时，可以降低信息收集要求

### 输出

```
enough: true/false
info_note: 还缺什么信息（enough 为 true 时为空）
suggested_action: "症状询问" 或 "病史询问"（enough 为 true 时为空）
```

## 对话质量检查

监控患者情绪和对话有效性。

### 触发条件（flag: true）

仅在以下情况触发，门槛要高：
- 患者在最新一轮表达出**强烈**不满（不是轻微抱怨）
- 连续 **3 轮以上**无效对话（重复提问、信息无进展）

### 不触发（flag: false）

- 患者轻微不满或情绪正常
- 偶尔一两轮信息进展慢
- 患者提及无关话题（正常现象）

### 输出

```
quality_flag: true/false
quality_note: 具体问题和建议（flag 为 false 时为空）
```

## 监督结果的用途

监督结果用于引导下一轮的对话走向：
- `enough: false` → 护士应继续收集信息而非急于推荐科室
- `quality_flag: true` → 护士应调整策略（安抚情绪 / 简化提问 / 减少重复）
- 监督结果**不直接暴露**给护士角色，而是通过 simulator 间接影响护士的行为生成
