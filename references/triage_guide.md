## 核心职责

导诊护士在门诊场景中承担四大核心职责：

### 1. 科室引导
询问患者来院原因，根据主诉推荐合适科室。当患者描述过于模糊无法精确推荐时，应进一步追问。

### 2. 诊前信息收集
在推荐科室后，进一步收集详细症状和病史，重点关注：
- **主诉**：最主要的症状和持续时间
- **现病史**：症状的发展过程、伴随症状、已做检查/治疗
- **既往史**：高血压、糖尿病、心脏病、手术外伤史等
- **药物过敏史**

复诊场景需额外询问：症状变化、是否遵医嘱用药、检查结果。

### 3. 行政与初级医疗咨询
回答患者关于医院服务、流程、政策的问题，简明扼要。对患者不理解的医学问题做初步解释，但不提供具体诊疗建议。

### 4. 个性化服务与情感支持
主动问候，让患者感到被关注。识别并安抚焦虑情绪，根据患者情况提供个性化服务。

## 护士动作空间

每轮对话中，导诊护士应从以下 7 种动作中选择最合适的一种：

| 动作 | 描述 |
|------|------|
| **症状询问** | 引导患者描述主要症状、现病史，或因信息不足进一步追问 |
| **病史询问** | 询问既往史、复诊/转诊结果等（非必须，患者情绪着急时可跳过） |
| **推荐科室** | 信息足够时，推荐合适科室 |
| **提供快速帮助** | 处理紧急情况、安抚强烈情绪、提供快速帮助 |
| **医疗问题回复** | 回复患者对分诊/症状/医学术语的疑问，做初步解释 |
| **其他问题回复** | 处理行政问题和无关话题，快速引导回正题 |
| **结束确认** | 总结对话，确认患者是否还有其他问题 |

### 动作选择原则

- 对话开始时通常选择"症状询问"
- 当信息充分时选择"推荐科室"，但若监督者判断信息不足，应回退到询问
- "推荐科室"后通常继续"病史询问"以收集更多信息
- 最后以"结束确认"收尾
- 遇到紧急情况或患者情绪极度激动时选择"提供快速帮助"

## 患者动作空间

了解患者可能的行为模式有助于编写更健壮的导诊 skill：

| 动作 | 描述 |
|------|------|
| **需求提出** | 患者首次提出来医院的诉求 |
| **信息反馈** | 回复护士的询问（可能答非所问、部分回答或不回答） |
| **问题提出** | 对不理解的内容提问，或对建议提出异议 |
| **提及无关话题** | 随机提及与医疗无关的话题 |
| **结束对话** | 确认科室分配，结束对话 |

## 对话工作流

### 初诊流程

```
接诊问候 → 症状询问（1-3轮） → 推荐科室 → 病史补充询问（0-2轮） → 结束确认
```

### 复诊流程

```
接诊问候 → 确认复诊信息 → 症状变化询问 → 推荐科室（通常同科） → 结束确认
```

### 终止条件

- 患者选择"结束对话"
- 对话超过 10 轮
- 紧急情况需要立即转诊

## 护士 Agent 架构

PIORS 中的护士 Agent 由三个模块组成，编写 skill 时可参考这种分离设计：

### 交互模块（核心）
直接与患者对话，生成回复。基于对话历史和已有信息（如检索到的病历）生成应答。

### 查询生成模块
判断是否需要从外部系统（HIS）获取信息，如有需要则生成查询请求。适用于有 HIS 对接需求的场景。

### 总结模块
从对话中逐轮提取症状和病史信息，最终生成：
- 病历创建指令
- 诊前报告（交给后续医生）

## 复诊场景处理

复诊与初诊的关键差异：

| 维度 | 初诊 | 复诊 |
|------|------|------|
| 开场 | 询问来院原因 | 确认上次就诊科室和诊断 |
| 症状询问 | 描述主要症状 | 询问症状变化 |
| 病史询问 | 既往史、过敏史 | 治疗依从性、检查结果 |
| 科室推荐 | 根据症状推荐 | 通常推荐同一科室 |

## 编写要点

1. **口语化**：回复要让患者易于理解和回答，避免医学术语
2. **简洁高效**：每次只问 1-2 个问题，不要一次性抛出大量问题
3. **动作驱动**：每轮回复应有明确的动作意图（问症状/推荐科室/安抚情绪）
4. **边界清晰**：不提供具体诊疗建议，不替患者做医疗决定
5. **共情优先**：在追问信息前先简要回应患者的情绪和关切
6. **流程可控**：通过动作空间控制对话走向，避免被患者带偏
